## ✨ 이번 디벨롭으로 들어간 핵심 메커니즘

**선택지 시스템 (4지선다 재설계)**
- 모든 시나리오가 기존 상/중/하 3개 → **균형형(8점) 1개 추가로 4지선다**.
- **확률형 보기(도박)**: 일부 「즉효」 보기는 성공/실패가 갈리는 도박. (예: 지하수 무제한 양수, 남획, 동토 시추, 지구공학 강행)
- **지연 청구서**: 값싼 선택이 지금은 멀쩡해도 N턴 뒤 청구서로 폭발. (석탄 재가동·매립지 확장·화학비료 등)
- **티핑포인트 결합**: 생태계가 낮을수록 기온 피해가 비선형으로 증폭.

**내러티브**
- 시나리오마다 **피해자의 목소리(화자)** 한 줄.
- **상태 인지형 콜백**: 과거의 선택(석탄·외면·무모한 도박)이 이후 결과 화면에 회상으로 되돌아옴.

**UX 연출**
- **선택 품질 즉시 판정**: 매 선택 직후 결과 화면 맨 위에 🟢최선 / 🟡절충 / 🟠아쉬운 균형 / 🔴값비싼 선택 배지 + 획득 점수 + **이번 턴 실제 지표 변화(🌡️·🌊·🌱, 좋으면 초록·나쁘면 빨강)** 를 띄워 「내가 좋은 선택을 했는지」를 그 자리에서 보여줍니다.
- **인트로 특수 모드 가이드**: 시작 화면의 접이식 카드에서 예산·정치 모드와 **⚡오버차지(상한 초과 비축)** 메커니즘을 설명합니다.
- 본문 **타이핑 효과** → **선택지 순차 등장(읽기 강제)** → **커밋 애니메이션**.
- 기온이 오를수록 **화면 전체가 붉게 달아오르는 색온도 드리프트**.
- **하드코어 모드**: 선택 **시간압박 링**(시간 초과 시 최악의 선택 강제 집행).

**라이브옵스 (반복 플레이)**
- **엔딩 도감**(12종 수집), **숨겨진 「대전환」 엔딩**(완벽한 플레이에서만 등장).
- **시너지 업적**: 완벽주의 통제관(전 단계 최선), 운명의 도박사(도박 3회 성공).
- **🗓️ 데일리 챌린지**: 날짜 시드로 전 세계가 같은 판 — 친구와 점수 경쟁.

**난이도 4종**: 🔥도전 · ☠️하드코어 · 💰예산 · 🗳️정치
- **예산/정치 모드**의 전략적 타협: 4지선다는 **2개 소비 / 2개 자본 확보**로 갈립니다. 환경을 포기하는 하·균형형 선택지는 오히려 **예산·정치자본을 회복**시켜, "이번 턴은 친환경에 투자할까, 환경 내주고 곳간 채울까"를 매 턴 저울질하게 됩니다.

**🪙 자원 메타(예산·정치 고급 시스템)**
- **무한 오버차지**: 자원 보유 상한 철폐. 100% 초과 시 게이지가 **금빛(예산)·보랏빛(정치)으로 흐르는 「과충전」 연출** + `⚡OVERCHARGE 130%` 칩 + `+30억 초과 비축` 표기. (부족할 때만 붉은 경고 — 색 의미를 분리)
- **수치적 비대칭**: 소비가 환급의 약 2.8배 — 환급 보기를 여러 번 골라야 소비 보기 한 번을 지를 수 있습니다.
- **💰 예산 = 돈의 시간가치**: 남긴 예산에 매 턴 **복리 이자 8%**. 단, **기후 재난 복구비**(현재 예산 × 기온 × 5%)가 매 턴 차압돼 — 지구가 뜨거우면 200~300억을 쌓아도 매 턴 뭉텅이로 녹아 '저축 고인물'이 견제됩니다. + **블랙스완 시장 쇼크**(과대 축적 시 확률적 초인플레이션 증발).
- **🗳️ 정치 = 여론 줄타기**: 강한 규제는 지지율↓·인기영합은 지지율↑. 지지율로 정치자본이 충원되고, **임기 심사 커트라인이 3·6·9단계마다 35→55→75%로 급상승**. 방어막 둘: **콘크리트 지지층**(지지율 100% 초과 구간에선 소비형 규제의 지지율 페널티 절반) + **8점 균형안 인공호흡기**(균형형 선택 시 지지율 +4 수급) → 20점만 찍다 가랑이 찢어지는 대신 균형안을 섞어 3·6·9 장벽을 밀당.
- **후반 그리디 폭주**: 후반에 돈/표 벌려고 환경 포기 보기를 고르면 기온·생태 타격이 매섭게 가속됩니다.
- **⚠️ 패닉 버튼(판당 1회)**: 예산 「🏦 지구 저당 금융」(환경 담보 긴급 예산) / 정치 「📢 우민화 정책」(지구를 불태워 지지율 매수) — 파산·불신임 직전의 비상 탈출구.
- **📜 개인화 엔딩 크로니클**: 당신이 고른 선택의 궤적(석탄 재가동·도박·외면·대전환 등)을 엔딩에서 한 편의 통치 기록으로 서술합니다.

## 🚀 GitHub Pages 배포 (3단계)

> 이 폴더는 push만 하면 자동 배포되도록 GitHub Actions 워크플로우(`.github/workflows/deploy.yml`)가 들어 있습니다.

### 1) GitHub에 새 저장소를 만들고 코드를 올립니다
이 폴더에서 터미널을 열고 아래를 그대로 실행하세요:

```bash
rm -rf .git          # (있다면) 동봉된 git 메타데이터를 깨끗이 비우고 새로 시작
git init
git add .
git commit -m "feat: 선택지 시스템 디벨롭(4지선다·도박·지연·도감·데일리·예산/정치)"
git branch -M main
git remote add origin https://github.com/<your-username>/<repo-name>.git
git push -u origin main
```

> 첫 줄 `rm -rf .git` 은 안전합니다 — 작업 파일(index.html·main.js·data.js)은 그대로 두고
> git 이력만 초기화해 깨끗한 첫 커밋을 만듭니다.

### 2) Pages 소스를 GitHub Actions로 지정
저장소 → **Settings → Pages → Build and deployment → Source** 를 **GitHub Actions** 로 선택.

### 3) 끝
push가 끝나면 **Actions** 탭에서 배포가 돌고, 1~2분 뒤
`https://<your-username>.github.io/<repo-name>/` 에서 게임이 열립니다.
이후엔 코드를 고쳐 `git push` 할 때마다 자동 재배포됩니다.

### (선택) GitHub CLI가 있다면 한 번에
```bash
gh repo create <repo-name> --public --source=. --remote=origin --push
```
그 다음 Settings → Pages → Source를 **GitHub Actions** 로 바꿔주세요.

## 🧱 시나리오 확장 파이프라인 (구조 동결 / Structure Freeze)

데이터 계약이 **동결**되어 있어, 아래 형식만 지키면 시나리오를 몇 개든 추가해도 `main.js`는 그대로 동작합니다. `node test.js`가 계약 위반을 즉시 잡아냅니다(게이트키퍼).

**1) 기본 시나리오** — `data.js`의 `SCENARIOS.{early|mid|late|final}` 배열에 추가:
```js
{ title:'고유 제목', text:'상황 설명', art:'heat',   // art는 main.js SCENE_ART의 키여야 함
  voice:'— 화자: "대사"',                            // (선택) 피해자 목소리
  choices:[
    //  C(label, tag, temp, sea, eco, score, sdg, feedback, fact, opts?)
    C('상(최선) 보기','무탄소·고비용', -0.05,0,+8, 20, 7, '결과 한 줄', '과학 팩트'),  // ★ score 20 = 시나리오당 정확히 1개
    C('중(절충) 보기','즉효·부담',     +0.05,0,+3, 10, 7, '결과', '팩트'),
    C('하(대가) 보기','저비용·청구서', +0.30,+4,-10, 6, 7, '결과', '팩트',
       { delay:{after:2,fx:{temp:+0.1},msg:'…'}, gamble:{p:0.5,win:{eco:+5},lose:{eco:-12}}, flag:'coal_on' }),
  ]},
```
**2) 4번째 「균형형(8점)」 보기** — `EXTRA_CHOICES`에 **제목을 키로** 1개 필수(없으면 테스트 실패):
```js
'고유 제목': EX('균형형 보기','균형형·설명', -0.02,0,+5, 8, 7, '결과', '팩트'),
```

**동결 계약(테스트가 강제):** ① 시나리오마다 `score 20` 보기 **정확히 1개** ② 모든 시나리오에 `EXTRA_CHOICES`의 `score 8` 보기 1개 ③ `art` 키가 `SCENE_ART`에 존재 ④ `sdg`가 `SDG`에 존재 ⑤ 제목 중복 금지 ⑥ `secret` 보기는 `req` 필수.
→ 예산/정치 비용, 4지선다 셔플, 엔딩 분기는 **전부 자동**으로 새 시나리오에 적용됩니다.

> 큐 인출은 `early2·mid3·late3·final2 = 10단계`. 그룹별 보유량이 인출 수 이상이면 풀이 커질수록 **매판 다른 조합**이 나와 반복 플레이성이 올라갑니다.
